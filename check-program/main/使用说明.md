# 策略一主程序使用说明

本说明介绍如何运行 **策略一** 主程序（`build_strategy1.py`），对科技/数字/养老产业贷款台账进行行业代码匹配核查，以及如何配置 `config.xlsx`（含 **clue 线索规则表**）。

---

## 一、工具做什么

- **输入**：机构提供的贷款台账（Excel）、参照表、国民经济行业分类映射表。
- **输出**：带核查结果的 Excel（`策略一核查结果_时间戳.xlsx`），在原台账右侧按产业类别追加报送/匹配/疑似正确/错报/漏报/疑似漏报、是否线索、备注等列，便于人工复核。

支持三类台账：**科技产业**、**数字经济产业**、**养老产业**。

### 术语（必读）

| 简称     | 全称                                       | 在 config.xlsx 中的位置 | 用途 |
|----------|--------------------------------------------|-------------------------|------|
| **参照表** | 五篇大文章与国民经济行业分类对应参照表       | **mapping 工作表 B1**   | 行业与科技/数字/养老产业代码对应；用于匹配与错报/漏报等列的【code:原始映射】展示 |
| **映射表** | 国民经济行业分类映射表                       | **mapping 工作表 B2**   | 行业小类代码与描述对应；用于结果中的“行业小类描述”列 |

- **config 的 mapping 工作表**：指 `config.xlsx` 里名为 `mapping` 的 sheet，用于配置参照表路径（B1）和映射表路径（B2），与上述“映射表”文件是两回事。

---

## 二、运行前准备

1. **Python 环境**：Python 3，已安装 `openpyxl`、`pandas` 等依赖。
2. **文件就位**：
   - **参照表**：五篇大文章与国民经济行业分类对应参照表（.xlsx），路径在 mapping **B1**。
   - **映射表**：国民经济行业分类映射表（.xlsx），路径在 mapping **B2**。
   - **台账文件**：待核查的科技/数字/养老产业贷款明细表（.xlsx），可放在 `main` 目录或子目录（如 `伪数据`），或通过 mapping 中“源文件/源文件路径”指定。
3. **首次运行**：若本目录下没有 `config.xlsx`，程序会提示是否新建并初始化；选 `y` 会生成空 config，并根据扫描到的台账自动补全配置行。

---

## 三、如何运行

在 **check-program/main/** 目录下执行：

```powershell
python build_strategy1.py
```

或在项目根目录：

```powershell
python check-program/main/build_strategy1.py
```

运行后进入**交互菜单**，根据提示输入序号并回车。

---

## 四、菜单功能说明

| 序号 | 功能 |
|------|------|
| **1** | **执行策略一**：按 `config.xlsx` 和 mapping 中的路径读取参照表、映射表和台账，按 **clue 表**规则生成核查结果 Excel，并在 config 的 log 表追加当次运行记录。 |
| **2** | **配置参照表路径**：将参照表路径写入 mapping 表 **B1**。 |
| **3** | **批量配置源文件路径**：在 mapping 表中维护“源文件 → 源文件路径”列表，支持台账不在本目录时的绝对路径。 |
| **4** | **配置映射表路径**：将映射表路径写入 mapping 表 **B2**（用于“行业小类描述”列）。 |
| **5** | **重置 config 工作表**：若存在 `config.xlsx`，则保留 **config 表、clue 表整表**及 **log 表头**写回；若无则按当前 mapping 源文件或本目录台账推断 config 行，clue 表使用内置 12 条规则。 |
| **6** | **执行策略一（备用）**：与选项 1 相同逻辑（均读取 clue 表），作为备用入口。 |
| **0** | **退出**。 |

日常使用：先通过 **2、3、4** 配好参照表、台账、映射表路径，再选 **1** 执行策略一；若台账或表结构变更，可用 **5** 重置 config 后重新执行。

## 五、执行结果说明

- **结果文件**：在 `check-program/main/` 下生成 `策略一核查结果_YYYYMMDD_HHMMSS.xlsx`。
- **结果内容**：每个台账对应结果簿中的一张 sheet，原表右侧按产业类别追加以下列（每个产业 **9 列**），最后一列为“行业小类描述”。

### 每个产业 9 列

| 列名 | 说明 |
|------|------|
| `报送-<类别名>` | 机构报送的该产业分类代码 |
| `匹配-<类别名>` | 参照表中该行业可匹配的产业代码；带*或带原始映射的以【code*:原始映射】形式展示 |
| `疑似正确-<类别名>` | 当 clue 规则主标签判为「疑似正确」时，展示匹配到的代码（格式同匹配列），否则为空 |
| `错报-<类别名>` | 机构多报或疑似多报的合并：机构报了但参照表不存在的代码【code】、以及机构报了且匹配到带*的代码【code*:原始映射】；多段用；分隔 |
| `漏报-<类别名>` | 参照表该产业下非*代码未报，格式【code:原始映射】 |
| `疑似漏报-<类别名>` | 参照表该产业下带*代码未报，格式【code*:原始映射】 |
| `是否线索-<类别名>` | 该行该产业依 clue 规则或主标签判定为线索时填“是” |
| `是否疑似线索-<类别名>` | 该行该产业依 clue 规则或主标签判定为疑似线索时填“是” |
| `备注-<类别名>` | 触发的 clue 规则编号及该条规则的备注；单条为 `触发规则编号 1：备注内容`，多条命中时用「；」拼接；未匹配时为空 |

示例（科技台账）：每个产业顺序为 `报送-高技术制造业`、`匹配-高技术制造业`、`疑似正确-高技术制造业`、`错报-高技术制造业`、`漏报-高技术制造业`、`疑似漏报-高技术制造业`、`是否线索-高技术制造业`、`是否疑似线索-高技术制造业`、`备注-高技术制造业`，随后为高技术服务业、战略性新兴产业、知识产权密集型产业同理；数字/养老台账则各 9 列对应数字经济产业或养老产业。

### 最后一列

- **行业小类描述**：根据本行“贷款投向行业”的 4 位行业小类代码，从**映射表**的 J 列、L 列取值，按 `行业代码：描述内容` 的形式输出。例如 J 列为 `C2345*`、L 列为 `某某行业`，则结果列显示为 `C2345*：某某行业`（描述已去多余空格和换行）。映射表路径在 mapping **B2** 或菜单 4 配置。

### 运行记录

每次选 1 执行后，会在 `config.xlsx` 的 **log** 表中追加一行，包含运行时间、参照表路径、结果文件、各 sheet 的**错报/漏报/疑似漏报**数量汇总（格式为“产业名：笔数”，多产业用分号连接）。

### 核查逻辑简述

- **关联逻辑**：从“贷款实际投向行业”列提取 4 位行业代码（如 `C3951`），以该代码与参照表对应工作表匹配；根据台账类型（科技/数字/养老）及 config 中“参照表工作表序号”选择参照表 sheet。
- **结果列与标记**：错报列 = 机构报了但参照表该行业该产业下不存在的代码【code】+ 机构报了且匹配到带*的代码【code*:原始映射】；漏报 = 参照表该产业下非*代码未报【code:原始映射】；疑似漏报 = 参照表该产业下带*代码未报【code*:原始映射】。“原始映射”来自**参照表**的 U 列（可在 config“参照表原始映射列序号”中修改），与**映射表**（国民经济行业分类映射表）无关。主标签、是否线索等由 **clue 表**规则决定，见第七节。

## 六、配置文件 config.xlsx

### 6.1 config 工作表

每行对应一个台账。第 5 行为说明行，程序会忽略。

| 字段 | 说明 |
|------|------|
| 输出工作表名称 | 结果文件中该台账对应的工作表名称；留空则该行配置被忽略。 |
| 来源台账文件 | 来源台账路径（相对项目根目录或绝对路径）；**置空则该行被跳过**（控制台打印「跳过：来源台账文件为空」）；路径错误或文件不存在则该行被跳过。 |
| 台账类型 | **必须**为“科技 / 数字 / 养老”之一。脚本据此推断默认参照表 sheet、报送列-类别映射等；填其他值则该行整行跳过。 |
| 表头行号 | 台账表头所在行（数字）。 |
| 数据起始行号 | 数据起始行（数字），如表头在第 3 行、数据从第 4 行则填 4。 |
| 贷款投向行业列序号 | 该列单元格应包含类似 `C2345` 的行业码，程序用正则提取；列号填错或无数值则整行不产生结果。 |
| 机构报送产业分类列序号 | 机构填报的产业分类列号，多个用逗号分隔；写少会漏判，写错会把无关列当报送。 |
| 报送列-类别映射 | 如 `17:HTP,20:HTS,23:SE,26:PA`；映射错位会导致该列报送归入错误产业。 |
| 参照表工作表序号 | 参照表使用的工作表序号（从 1 开始）或工作表名称（需与参照表内 sheet 名完全一致）。 |
| 参照表产业分类代码列序号 | 参照表中产业分类代码列号。 |
| 参照表行业4位码列序号 | 参照表中行业 4 位代码列号。 |
| 参照表星标列序号 | 参照表中“是否标*”列号。 |
| 参照表原始映射列序号 | 参照表中“原始映射内容”列号（默认 21 即 U 列），用于错报/漏报等列的【code:原始映射】。 |

### 6.2 mapping 工作表（路径配置）

此处“mapping”指 **config.xlsx 中的工作表名称**，勿与“映射表”（国民经济行业分类映射表）混淆。

| 单元格 | 含义 |
|--------|------|
| A1/B1 | 表头“参照表路径” / **参照表**（五篇大文章…）的文件路径。 |
| A2/B2 | 表头“映射表路径” / **映射表**（国民经济行业分类映射表）的文件路径，用于“行业小类描述”列。 |
| 下方 | 可选“源文件”“源文件路径”两列，其下为源文件名称与路径对应。 |

可通过菜单 2、4 分别配置 B1、B2。

### 6.3 log 工作表（运行日志）

每次执行策略一后追加一行，包括：运行时间、参照表路径、结果文件、明细 sheet、来源台账、台账类型、使用的参照表 sheet；**错报数量**、**漏报数量**、**疑似漏报数量**（格式为“产业名：笔数”，多产业用分号连接）。

### 6.4 clue 表

线索规则表，见第七节。

### 注意事项（config / 运行环境）

- 本脚本依赖 `openpyxl`，请确保环境可用。
- 若程序根目录下没有 `config.xlsx`，运行时会提示是否新建并初始化；选 y 将创建含 config 表头、mapping（A1/B1、A2/B2）、log 表头、clue 默认 12 条的 config，若有台账则自动补全 config 数据行。
- 类别代码与中文名对应：`HTP=高技术制造业`、`HTS=高技术服务业`、`SE=战略性新兴产业`、`PA=知识产权密集型产业`、`DE=数字经济产业`、`EC=养老产业`。
- 若台账列位置或表头变化，请修改 config 工作表中对应列序号。结果文件若被 Excel 打开，保存时会自动另存为带时间戳的新文件。

## 七、clue 表（线索规则表）读取与使用

### 7.1 从哪里读、何时读

- **文件**：`config.xlsx`（与 `build_strategy1.py` 同目录）。
- **工作表**：名为 **`clue`** 的 sheet（英文小写）。
- **时机**：每次选择菜单 **1. 执行策略一** 时，在开始处理台账前会**读一次** config，得到当次的 `clue_rules`，中间不会再读。

改完 clue 后要**保存 config.xlsx**，再重新选 1 执行策略一，新配置才会生效。

### 7.2 列顺序（共 9 列，固定按列号读）

程序**按列号**读 clue，**不按表头名**。列顺序必须与下表一致：

| 列号 | 含义           | 程序读取的单元格示例 |
|------|----------------|----------------------|
| 1    | 编号           | 仅作展示，程序未用   |
| 2    | 匹配到的产业数量 | 必须是 `0`、`1` 或 `>1` |
| 3    | 行业是否含星   | `是`、`否`、`-` 或 `否/是` |
| 4    | 机构报送产业为空? | `是`、`否` 或 `-` |
| 5    | 命中情况       | `是`、`否` 或 `-` |
| 6    | **主标签**     | 正确 / 疑似正确 / 漏报 / 疑似漏报 / 多报 / 错报 |
| 7    | **是否线索**   | `是` 或 `否`，控制结果表中「是否线索-xx产业」列 |
| 8    | **是否疑似线索** | `是` 或 `否`，控制结果表中「是否疑似线索-xx产业」列 |
| 9    | 备注/副标签    | **不参与逻辑判断**；程序会读取并写入结果表「备注-产业名称」列（单条或多条命中时拼接） |

- **第 3 列「行业是否含星」**：填 **`-`** 与填 **`否/是`** 在**程序逻辑上效果相同**（都表示“不区分含星或不含星，本条规则都匹配”）。语义上可区分：`-` 更偏“不关心”，`否/是` 更偏“两种都可以”。
- 第 1 行为表头，**从第 2 行开始**为规则数据。若某行**第 6 列（主标签）为空**，该行会被跳过。若所有行主标签都为空或 clue 表不存在，程序改用**内置默认 12 条规则**。

### 7.3 程序如何判「是」/「否」/「-」

程序对**每一行台账、每一个产业**先算出下列条件，再与规则表逐条比对：

| 列 | 含义 | 程序何时判为「是」 | 程序何时判为「否」 | 规则里填「-」或「否/是」 |
|----|------|-------------------|-------------------|--------------------------|
| 2 | 匹配到的产业数量 | 转成 `0`/`1`/`>1` 与规则比对 | 同上 | 不适用 |
| 3 | 行业是否含星 | **机构在本产业下报送的代码**里，至少有一个落在「本行贷款投向行业+当前产业」在参照表中的**带*候选**里 | 否则（或机构未报该产业） | 不区分含星不含星（**填 `-` 或 `否/是` 效果相同**） |
| 4 | 机构报送产业为空? | 本行该产业**没有**报送任何代码 | 本行该产业**有**报送至少一个代码 | 不区分空/非空 |
| 5 | 命中情况 | 报送的代码里**有**落在「匹配到的产业」候选里的 | 报送的代码里**没有**落在候选里的 | 不区分命中与否 |

「行业是否含星」与**机构报送**绑定：只看机构在本产业下实际报送的代码中，是否含有该行业、该产业在参照表里带*的候选。

**填法建议**：第 4 列只在「机构没报该产业」时适用填**是**，只在「机构报了」时适用填**否**，不区分填**-**。第 5 列只在命中时适用填**是**，只在未命中时适用填**否**，不区分填**-**。第 7、8 列直接控制结果表「是否线索」「是否疑似线索」；留空时程序按主标签推导。

### 7.4 匹配规则

1. 规则列表由 clue 第 2～8 列有效行（主标签非空）按**行顺序**组成；第 9 列仅用于结果表备注展示，不参与逻辑。
2. 对每行台账、每个产业，程序算 5 个条件（匹配数量、行业是否含星、报送空、命中、是否有额外报送；从 Excel 加载时不区分「混报」）。
3. 从规则表**第一条**开始，找到**第一条**条件都满足的规则，即用该条的**主标签、是否线索、是否疑似线索**；若有**多条**命中，**主标签**等仍取第一条，**备注**列会拼接所有命中规则的编号与备注（`触发规则编号 1：备注1；触发规则编号 2：备注2`）。
4. 是否线索/是否疑似线索：优先用规则第 7、8 列；若留空则按主标签推导（漏报/多报/错报→线索，疑似漏报/疑似多报/疑似错报/疑似正确→疑似线索）。

**行顺序很重要**：同一组条件可能被多条规则匹配，只有第一条会决定主标签与线索。

### 7.5 改了配置结果没变（排查清单）

1. **是否保存了 config.xlsx？** 改完 clue 后要在 Excel 里保存，再运行策略一。
2. **运行的是不是这个 config？** 程序只认 **`build_strategy1.py` 所在目录下的 `config.xlsx`**。
3. **clue 表是否为 9 列、列顺序是否一致？** 列数或顺序错位会导致主标签读到空，整表被跳过，改用内置 12 条规则。
4. **主标签列是否都有内容？** 第 6 列为空的行会被跳过。
5. **是否重新执行了策略一？** 改 config 后需**再选一次 1** 跑一遍。

### 7.6 如何确认当前用的是不是你的配置

执行策略一时看终端：

- **“已从 config 加载 clue 规则：N 条”** → 用的是你 config 里 clue 的 N 条规则。
- **“已从 config 加载 clue 规则：0 条，使用内置默认规则”** → clue 没读到有效行（列错位或主标签全空），在用内置 12 条。

看到 0 条时，请对照 7.2、7.5 检查 clue 的**列顺序**和**主标签列**。

## 九、常见问题

- **没有任何错报/漏报/疑似漏报结果**：可能台账与参照表一致；或产业分类列为空/格式不一致；或 config 中列序号与当前台账结构不符。检查 config 列序号及“报送列-类别映射”、参照表路径（mapping B1）。
- **没有生成结果文件**：确认执行时选择了菜单 **1**；若报错，看终端提示（如缺少 config、参照表不存在等）。
- **行业小类描述列为空**：确认**映射表**（国民经济行业分类映射表）文件存在、路径在 mapping **B2** 正确，且该表含与台账行业代码对应的小类代码列（默认 J 列）和描述列（默认 L 列）。
- **如何重新跑一版**：更新台账或参照表后，运行 `python build_strategy1.py`，选 **1** 执行策略一，会覆盖或另存新的 `策略一核查结果_时间戳.xlsx`。
- **改了 clue 没效果**：见第七节 7.5、7.6（保存 config、列顺序 9 列、主标签非空、重新选 1 执行）。
